# Technical Interview - Senior Full-Stack Developer

## Overview

We want to assess your ability to develop a small web application from scratch, making architectural, technical, and design decisions on your own.

Your goal will be to create a **note-taking** application that allows users to **create, edit, organize, and search** their notes effectively. This test is designed to evaluate both your technical knowledge and your approach to development, code structure, analytical skills, and user experience.

---

## General Requirements

### 📋 Expected Features

- Users must be able to create, update, and delete notes.
- Notes must be organizable using some form of classification (e.g., categories, tags, or folders).
- It must be possible to search for or filter notes efficiently.
- The interface must be clear, functional, and usable on different screen sizes.
- The application must display loading indicators and handle errors appropriately.

You are free to decide how to meet these requirements: which database structure to use, how to model relationships, which endpoints or actions to implement, etc.

---

## Development Environment

### ⚙️ Suggested Stack

We recommend using technologies similar to those we use in our real-world work environment:

- Next.js 15+
- TypeScript

So we've prepared a minimal scaffolding app under `notes-app`, but if you’d rather use a different stack that you’re comfortable with and that you think will best showcase your skills, that’s fine. We’re more interested in your reasoning and execution than in whether you use a specific stack.

---

## We’ll pay special attention to

1. **Code quality**
    - Clarity, structure, typing, and organization
    - Effective use of modern tools and design patterns
    - Validations and error handling
2. **Design and architecture**
    - Ability to model the domain well (notes, grading, etc.)
    - Well-reasoned decisions regarding folder structure, abstractions, components, etc.
3. **User experience**
    - Intuitive and aesthetically pleasing interface
    - Overall fluidity and user feedback
    - Responsive design
4. **Analytical skills**
    - How you interpret the given requirements
    - How you break down features into tasks
    - What decisions you make and how you justify them

---

## Estimated time

Time is not a key factor. We recommend spending about an afternoon on this, though you can take more or less time if needed. There is no strict time limit.

---

## Submission

1. Work in a new branch using your name as branch's name, using `main` branch as base from this GitHub repository.
2. The application scaffold lives in the `notes-app/` directory — build your solution there.
3. Include a brief `INSTRUCTIONS.md` file containing:
    - Installation and execution instructions
    - Stack and tools used
    - Description of the project architecture
    - Key technical decisions and why you made them
    - You will use this document for the discussion later about the solution
4. Push your branch.

---

## Final Considerations

You’re not expected to complete all the features. What we value most is **how you reason and execute** the tasks you do tackle. In the follow-up interview, we’ll discuss your decisions, what you prioritized, and why.

**Good luck, and thank you for your interest in Qalea!**
